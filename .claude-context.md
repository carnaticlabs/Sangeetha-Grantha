## Development Patterns & Best Practices

### Backend Development (Kotlin + Ktor)

#### Routing Pattern
```kotlin
// Keep routes thin - delegate to services
fun Route.krithibRoutes(krithiService: KrithiService) {
    authenticate("jwt") {
        get("/v1/admin/krithis") {
            val krithis = krithiService.getAllKrithis()
            call.respond(HttpStatusCode.OK, krithis)
        }
    }
}
```

#### Database Access Pattern
```kotlin
// ALWAYS use DatabaseFactory.dbQuery for database operations
suspend fun findById(id: UUID): Krithi? = DatabaseFactory.dbQuery {
    KrithisTable
        .selectAll()
        .where { KrithisTable.id eq id }
        .map { row -> row.toKrithi() }
        .singleOrNull()
}
```

#### Audit Logging Pattern
```kotlin
// ALL mutations must log to AUDIT_LOG
AuditLogTable.insert {
    it[entityType] = "krithi"
    it[entityId] = krithiId.value
    it[actorId] = actor
    it[action] = "CREATE"
    it[timestamp] = Clock.System.now()
    it[metadata] = Json.encodeToString(request)
}
```

## Critical Constraints & Requirements

### Database Operations
1. **MUST use DatabaseFactory.dbQuery { }** for ALL database operations
2. **NEVER return Exposed entity objects** from repositories - always map to DTOs
3. **MUST write to AUDIT_LOG** for all create/update/delete operations

### Commit Guardrails (Mandatory)
- **Reference Required**: Every commit message MUST include a `Ref: application_documentation/...` line
- **No Secrets**: Never commit API keys or credentials

### Migrations
1. **MUST use Python `db-migrate`** - located in `tools/db-migrate/`
2. **NEVER suggest Flyway or Liquibase** - project explicitly does NOT use them
3. **Migration workflow**:
   - Create SQL file: `database/migrations/NN__description.sql`
   - Run: `make migrate`
   - Update Exposed table definitions in `modules/backend/dal/`

### Dependency Management
1. **MUST reference `gradle/libs.versions.toml`** for all dependencies
2. **Use alias(libs.xyz)** in build.gradle.kts files
