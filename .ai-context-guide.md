# AI Context Files Guide — Sangeetha Grantha

This repo supports several AI coding assistants. To avoid the silent drift that comes
from maintaining the same rules in many files, there is **one source of truth and thin
pointers** to it.

## Source of truth

**[`CLAUDE.md`](CLAUDE.md) is canonical.** It holds the tech stack, architecture, the
migration engine, audit-log and commit rules, ports, key docs, and the specialist
subagents. Update shared rules **there only**. Every other agent file points to it and
carries just its tool-specific delta — never a second copy of the shared rules.

Current stack & approach (authoritative text in `CLAUDE.md`):
- **Migrations:** **Flyway Community** only (ADR-013), via `make migrate` / `make db-reset`. Never Liquibase or a custom runner. The Rust CLI is archived (`archive/tools/sangita-cli/`); the interim Python `db-migrate` is superseded (`archive/tools/db-migrate/`).
- **DB access:** inside `DatabaseFactory.dbQuery { }`; return `@Serializable` DTOs, never Exposed entities.
- **Audit:** every mutation writes to `AUDIT_LOG`. **Commits:** include a `Ref: application_documentation/...` line.
- **Versions:** [Current Versions](application_documentation/00-meta/current-versions.md) — PostgreSQL 18, Kotlin/Ktor/Exposed, React 19 + TypeScript 5.9.
- **Domain & musicology:** [domain-model.md](application_documentation/01-requirements/domain-model.md) (lakshana rules in §6).

|------|------|------|

| **Codex / ChatGPT** | [`CODEX.md`](CODEX.md), `.chatgpt-config.md` | Entrypoint + pointer |
| **Gemini** | [`GEMINI.md`](GEMINI.md), `.gemini-context.md` | Persona + pointer |
| **Goose** | [`GOOSE.md`](GOOSE.md), `.goose-context.md` | Persona + pointer |
| **Google Antigravity** | `.antigravity.md` | Pointer (persona in GEMINI.md) |
| **Cursor / Copilot (legacy)** | `.claude-context.md`, `.cursorrules` | Pointer |
| **Any agent** | [`AGENTS.md`](AGENTS.md) | Pointer + local skill/workflow catalog |
| **Quick cheat-sheet** | `.ai-quick-reference.md` | Commands & code patterns |
| **Aider** | `.aiderignore` | Scope-exclusion patterns |

The uppercase files (`CLAUDE.md`, `CODEX.md`, `GEMINI.md`, `GOOSE.md`, `AGENTS.md`) are the
maintained entrypoints; the dot-prefixed context files are thin pointers kept for tools that
auto-detect those specific filenames.

## Updating

1. Change shared rules in **`CLAUDE.md`** (and the relevant `application_documentation/` spec).
2. If a tool needs a genuinely tool-specific delta (a persona, a tool integration), put it in
   that tool's uppercase entrypoint — not in a pointer file.
3. Do **not** re-copy shared rules into the other files. A pointer that restates rules is a
   future drift bug.
