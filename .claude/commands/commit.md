Create a commit following this project's conventions.

Requirements:
1. Review all staged and unstaged changes
2. Draft a commit message that:
   - Summarizes the change concisely
   - Includes a `Ref: application_documentation/...` line pointing to the relevant spec
3. Stage relevant files (never stage .env, credentials, or secrets)
4. Create the commit

If unsure which documentation reference to use, check `application_documentation/` for the most relevant spec file.
